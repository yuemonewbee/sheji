## 🔴 P0 · 隐患修复（地基，优先）

- [x] **静态服务路径穿越漏洞**：`path.join(PUBLIC_DIR, req.url)` 可用 `../` 读到 public 外的源码。需解析后校验仍在 `PUBLIC_DIR` 内，否则 403。
- [x] **主循环无兜底**：`setInterval` 里 `updateWorld` 抛异常会整服崩、全员掉线。包 `try/catch`，出错跳过该帧并记录。
- [ ] **WS 消息无校验/限流**：服务器信任客户端消息频率。可加每连接的消息速率限制，防刷。
- [ ] **无最大人数上限**：无限连接会拖垮单房间，加 `MAX_PLAYERS` 守卫。

## 🟡 P2 · 网络 / 性能（接着带宽优化做）

- [x] **开启 WebSocket 压缩 `perMessageDeflate`**：一行配置，JSON 文本压缩率高，继续降带宽。
- [x] **二进制协议（ArrayBuffer）**：高频 state 快照改成二进制定长打包（单位/事件用数字 nid 引用，roster 带 id↔nid 映射），客户端 decode.js 还原成同形快照对象、render/hud 零改动。回归测试 19 万项字段全对、角度误差 <0.0006rad；体积约为压缩后 JSON 的 40%（且二进制还会再被压缩），另省 JSON parse 的 CPU/GC。其它消息仍 JSON。
- [ ] **子弹平滑**：现子弹不插值、15Hz 会跳。下发速度让客户端外推。
- [ ] **只发变化的实体**：心心/箱子/特效很少变，按需发而非每帧重打包。
- [ ] **tick 抗漂移**：`setInterval` 会累积漂移，改基于时间差的补偿循环。
- [ ] **复用快照缓冲**：`snapshot()` 每帧新建数组/对象，造成 GC，可对象池化。

## 🎮 可玩性方向（点子库）

**🎯 游戏模式（耐玩度最大杠杆，房主大厅选）**
- [x] **军备竞赛（Gun Game）**：每击杀升一级换下一把枪（梯长 7），走完即获胜；无拾取、无手雷；房主大厅可选模式。
- [x] **占点 / 中心控制（KOTH）**：中央占领圈，单队独占每秒 +1 分（敌我同在则争夺暂停），先到 60 秒获胜；击杀不计胜负分。圈颜色随控制方变化。
- [x] **夺旗（CTF）**：各队基地一面旗，碰敌旗即扛、带回己方旗位（且己方旗在家）得分；扛旗者阵亡旗掉落、可被夺回或 8 秒后自动归位；先夺 3 次获胜。
- [x] **电脑 AI 按模式分配意图**：占点→聚向圈中心（实测 4.4× 随机密度）；夺旗→抢旗/扛旗回家/追回被偷的旗/部分队员返还掉落旗；路上仍照常交火。团队死斗/军备竞赛保持纯打人。
- [ ] **最后存活**：回合制、不复活，最后站着的队赢。
- [ ] **限时赛**：固定时间内击杀最多的队赢。

**🟠 深度与爽感**
- [ ] **冲刺 / 闪避**：冷却技能，瞬间位移，增加走位深度。
- [~] **连杀奖励（Killstreak / Multikill）**：已做播报气氛——窗口内连杀(~4.5s)触发 Double/Triple/Quadra/Penta...Kill 语音(TTS)+合成音效、kill feed 大字「双杀!/三杀!」、头顶燃烧火焰特效(等级越高越大)。**待做**：增益效果（加速/伤害/护盾）。
- [ ] **增益拾取**：护盾/加速/双倍伤害等限时 buff，制造抢点博弈。
- [x] **出生保护**：出生/复活后 ~1.5s 无敌 + 青色护盾环 + 身体闪烁，开火即解除，防堵家喂分。

**🟡 低成本提味**
- [ ] **电脑难度选项**（简单/普通/困难）：调瞄准抖动/反应/射程。
- [x] **开局 3-2-1 倒计时**：开局冻结 ~3 秒（不能移动/开火），中央显示 3-2-1-GO；出生保护从开打瞬间起算。
- [ ] **赛事播报**：首杀 / 连杀 / 复仇，用 kill feed 滚动显示。
- [ ] **回合制（Bo3/Bo5）**：多回合定胜负。
