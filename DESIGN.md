# 网页多人射击游戏 —— 初步设计文档

> 一个 2D 俯视角、双方对战的网页小游戏，玩法参考 CS（移动、瞄准、射击、计分）。
> 目标：先在本地/局域网和几个朋友能玩起来。

---

## 1. 一句话玩法

两支队伍（红队 vs 蓝队）的玩家，在一张有障碍物（墙）的地图里跑动、瞄准、开枪，
击杀对方加分，比谁先到达目标分数（或一局时间内分高者胜）。

操作方式：
- **WASD** 控制角色上下左右移动
- **鼠标** 控制瞄准方向（角色"枪口"朝向鼠标）
- **鼠标左键** 开枪
- 子弹会被墙挡住，打中对方角色造成伤害，血量归零即"阵亡"，几秒后复活

---

## 2. 为什么这样选型（给零基础的解释）

| 选择 | 用什么 | 为什么 |
|------|--------|--------|
| 画面 | HTML5 **Canvas** + 原生 JavaScript | 浏览器自带，不用装引擎；2D 俯视角画圆/方块即可，最容易上手 |
| 联机 | **WebSocket**（`ws` 库） | 浏览器和服务器之间一条"实时电话线"，双方随时能互相发消息，适合实时游戏 |
| 服务器 | **Node.js** | 前端后端都用 JavaScript，一种语言走天下，零基础不用切换思维 |
| 不用框架 | 暂不用 React/Vue/游戏引擎 | 初期保持简单，先把原理跑通，理解了再考虑升级 |

## 3. 整体架构：服务器说了算（Server-Authoritative）

这是多人游戏最关键的概念，先理解它，后面都顺：

```
   玩家A的浏览器                服务器(Node.js)              玩家B的浏览器
  ┌──────────────┐           ┌─────────────────┐          ┌──────────────┐
  │ 1.发送"我的输入"│  ──────>  │ 2.汇总所有人输入  │ <──────  │ 1.发送"我的输入"│
  │  (按了W、开枪) │           │   计算新世界状态  │          │  (按了D)      │
  │              │           │   (位置/血量/子弹)│          │              │
  │ 4.把收到的    │  <──────  │ 3.把最新世界状态  │ ──────>  │ 4.把收到的    │
  │   状态画出来   │           │   广播给所有人    │          │   状态画出来   │
  └──────────────┘           └─────────────────┘          └──────────────┘
```

要点：
1. **浏览器(客户端)只做两件事**：把玩家的操作发给服务器；把服务器发回的状态画到屏幕上。
2. **服务器是唯一的"真相"**：所有位置、碰撞、伤害都由服务器计算，防止有人作弊。
3. **服务器循环(tick)**：服务器每秒大约更新 30 次世界状态，每次都广播给所有玩家。

> 类比：服务器像棋盘的裁判，玩家只是说"我想往这走"，最终棋子怎么动、谁吃谁，裁判说了算。

## 5. 核心数据长什么样（初版）

服务器内存里维护一份"世界状态"，大致是：

```js
worldState = {
  players: {
    "玩家id": { x, y, angle, hp, team, score, alive },
    ...
  },
  bullets: [
    { x, y, dx, dy, ownerId, team },
    ...
  ],
  scores: { red: 0, blue: 0 }
}
```

地图（墙）用一组矩形表示，移动和子弹都要和它们做碰撞检测：

```js
walls = [
  { x: 200, y: 150, w: 100, h: 20 },
  ...
]
```

## 6. 开发路线图（先做单机人机对战，再做联机）

策略：**先把核心玩法在单机做完（对手是机器人），最后再做联机改造。**
这样能把最难的网络部分推到最后，前期专注好玩的玩法，调试也简单。

为避免将来返工，代码从一开始就分成三块、互不缠绕：
- **world（游戏逻辑）**：算移动、碰撞、开枪、扣血、计分。单机/联机都一样，将来直接搬到服务器。
- **input（输入）** 与 **draw（渲染）**：单机和联机时分别替换即可。

里程碑：

- [x] **M1 环境搭建**：安装 Node.js，建项目，跑通能在浏览器打开的网页
- [x] **M2 画一个角色**：Canvas 上画出能用 WASD 移动、枪口朝鼠标的小圆点（纯本地）
- [x] **M3 地图与碰撞**：加入墙体，角色撞墙会被挡住
- [x] **M4 射击系统**：鼠标瞄准、左键开枪，子弹飞行、打中扣血、阵亡随机复活、框外倒计时列表
- [x] **M5 机器人对手**：AI 敌人会走位、绕墙、瞄准、开枪 —— **人机对战**（标准难度已定）
- [x] **M6 计分与胜负**：击杀加分、显示比分、先到 10 分获胜、可重新开始 →（**单机版完成 ✅**）
- [ ] **M7 联机改造**：把 world 逻辑搬到服务器，接入 WebSocket，开两个窗口真人对战
- [ ] **M8 打磨**：队伍系统、回合制、音效、出生点、延迟优化等

> 建议节奏：一次只推进一个里程碑，每步我都会解释"这段代码在干嘛"。

## 7. 暂不考虑（避免一开始就太复杂）

- 反作弊优化、网络延迟补偿（插值/预测）——先能玩，后优化
- 复杂美术、地图编辑器、武器系统

这些等核心玩法跑通后，再按需要逐步加。
